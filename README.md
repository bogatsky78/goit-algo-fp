# Завдання 1 — Однозв'язний список

**Файл:** [task1.py](task1.py)

Реалізація однозв'язного списку з такими операціями:

## Можливості

- `reverse()` — розвертає список на місці шляхом перекомутації вказівників вузлів. O(n) за часом, O(1) за пам'яттю.

**Файл:** [task5.py](task5.py)

Будує дерево мін-купи (використовуючи логіку Завдання 4) і візуалізує обходи в глибину та в ширину, забарвлюючи кожен вузол відповідно до порядку відвідування — темні відтінки для ранніх відвідувань, світлі — для пізніх.

- `visualize_traversal(root, fn, title)` — призначає кольори вузлам у порядку обходу, потім відображає повністю розфарбоване дерево на одному графіку.

# Завдання 6 — Жадібний алгоритм проти динамічного програмування (вибір їжі)

**Файл:** [task6.py](task6.py)

Два підходи до задачі рюкзака з обмеженим бюджетом: вибір страв з меню для максимізації загальної калорійності.

## Можливості

- `greedy_algorithm(items, budget)` — сортує страви за співвідношенням калорій до вартості (за спаданням) і жадібно вибирає кожну страву, що вписується у залишок бюджету. O(n log n) за часом, O(1) додаткової пам'яті. Швидкий, але не завжди оптимальний.
- `dynamic_programming(items, budget)` — будує двовимірну таблицю ДП `dp[i][b]` = максимальна калорійність при використанні перших `i` страв у межах бюджету `b`, потім відновлює вибрані страви зворотним трасуванням. O(n × budget) за часом і пам'яттю. Завжди знаходить глобальний оптимум.

## Використання

```bash
python3 task6.py
```

Приклад виводу (бюджет = 100):

```
Greedy algorithm (budget=100):
  Selected: ['cola', 'potato', 'pepsi', 'hot-dog']
  Total calories: 870
  Total cost: 80

Dynamic programming (budget=100):
  Selected: ['potato', 'cola', 'pepsi', 'pizza']
  Total calories: 970
  Total cost: 100
```

Жадібний алгоритм витрачає лише 80 і дає 870 ккал; ДП використовує весь бюджет і знаходить глобально оптимальні 970 ккал.
